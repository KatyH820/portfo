006 :File-Playground

This program can find and display the paths of all of the files in a directory or subdirectories, then take action on them. The program will take input from the console in the following format. First, the program reads input that determines which files are eligible to be found. This input can be specified in two ways: D or R, followed by a space, followed by the path to a directory. D stands for the directory, and it will take all the files in that directory but not the subdirectory and print them. R stands for recursive, and it will take all the files in that directory and other files in its subdirectory and print them. Second, the program will read input to filter the files that the program should take action on. There are five ways to write this input. All the files will be considered if the input is the letter A alone. If the input starts with the letter N and a file name(including the extension), it will find and consider the files with the same name. If the input starts with the letter E and a particular extension, it will find and consider the files with the same extension. If the input starts with T and a given text, the program will find a text file that contains the text. If the input begins with '>' or '<' and a number, then the search will be for files in the specified threshold. The program will print the remaining files. If there are no remaining files, there is no action to take, and the program ends. Finally, the program will take input that describes the action. There are three ways to write this input. If the input is the letter F alone, the program will print the first line of text from the file if it is a text file. If the input is the letter D by itself, it will make a duplicate copy of the file and store it in the same directory where the original resides, but the copy will have .dup appended to its name. If the input is the letter T alone, it will modify its last modified timestamp to be the current date and time.
